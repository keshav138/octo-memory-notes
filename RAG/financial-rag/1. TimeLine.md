## Project Overview — Financial RAG API

### Stack

|Layer|Tool|
|---|---|
|RAG Framework|LangChain|
|LLM|Groq (switchable)|
|Vector DB|ChromaDB|
|Embeddings|HuggingFace `all-MiniLM-L6-v2`|
|API|FastAPI|
|Monitoring|Prometheus + Grafana|
|Containerization|Docker + Docker Compose|
|CI/CD|Jenkins|
|IaC|Terraform|
|Cloud|AWS EC2|
|Data Source|SEC EDGAR|

---

### High Level Flow

```
User uploads PDF / provides SEC filing URL
        ↓
Document Loader (LangChain)
        ↓
Text Splitter (chunk the document)
        ↓
Embeddings (HuggingFace MiniLM)
        ↓
ChromaDB (store vectors)
        ↓
User sends query via /query endpoint
        ↓
Retriever (fetch top-k relevant chunks)
        ↓
Prompt Template + LLM (Groq)
        ↓
Response returned
```

---

### API Endpoints

|Endpoint|Purpose|
|---|---|
|`POST /ingest`|Upload PDF or SEC URL, process + store in ChromaDB|
|`POST /query`|Ask a question against ingested documents|
|`GET /health`|Service health check|
|`GET /metrics`|Prometheus metrics scrape endpoint|
|`DELETE /reset`|Clear ChromaDB collection|

### Broad Phases

1. **RAG pipeline** — loader → chunker → embeddings → ChromaDB → chain
2. **FastAPI wrapper** — endpoints around the pipeline
3. **Prometheus instrumentation** — metrics on inference latency, request count
4. **Dockerize** — app + ChromaDB + Prometheus + Grafana in compose
5. **Jenkins** — CI pipeline, build + push Docker image
6. **Terraform + AWS** — provision EC2, deploy
