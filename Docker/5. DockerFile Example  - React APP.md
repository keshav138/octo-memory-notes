## Docker Architecture – High-Level Overview

Docker uses a **client-server architecture** with four main components:

### 1. **Docker Client**  
- The primary way users interact with Docker (via `docker` CLI or API).  
- Sends commands like `build`, `pull`, `run` to the Docker daemon.  
- Can communicate with a local or remote daemon.

### 2. **Docker Daemon (`dockerd`)**  
- The background service that manages Docker objects: images, containers, networks, volumes.  
- Listens for Docker API requests and handles container lifecycle (start, stop, destroy).  
- Can also talk to other daemons to manage swarm services.

### 3. **Containerd** (low-level component inside daemon)  
- Responsible for actually running containers (using runc).  
- Handles image transfer, storage, and execution.

### 4. **Registry**  
- Stores Docker images (e.g., Docker Hub, private registry).  
- `docker pull` → downloads images from registry.  
- `docker push` → uploads locally built images.

### 5. **Underlying technologies**  
- **Namespaces** – isolate processes (PID, net, mount, UTS, IPC, user).  
- **Cgroups** – limit resource usage (CPU, RAM, disk I/O).  
- **Union filesystems** – enable layers and copy-on-write (overlay2, aufs).

## The 20% of Docker Commands You Use 80% of the Time

### 1. **`docker pull`** – Download an image from registry  
```bash
docker pull nginx
```

### 2. **`docker run`** – Create + start a container (most important)  
```bash
docker run -d -p 80:80 --name web nginx
```
Flags: `-d` (detach), `-p` (port map), `-v` (volume), `--rm` (auto-remove)

### 3. **`docker ps`** – List running containers  
```bash
docker ps -a   # include stopped containers
```

### 4. **`docker stop`** / **`docker start`** – Stop or start an existing container  
```bash
docker stop web
docker start web
```

### 5. **`docker rm`** – Remove container(s)  
```bash
docker rm web
docker rm -f web   # force remove even if running
```

### 6. **`docker images`** – List downloaded images  
```bash
docker images
```

### 7. **`docker rmi`** – Remove image(s)  
```bash
docker rmi nginx
```

### 8. **`docker exec`** – Run a command inside a running container (debugging)  
```bash
docker exec -it web bash
```

### 9. **`docker logs`** – View container’s stdout/stderr  
```bash
docker logs -f web   # follow live logs
```

### 10. **`docker build`** – Build an image from a Dockerfile  
```bash
docker build -t myapp:1.0 .
```

### 11. **`docker push`** – Upload image to registry  
```bash
docker push myrepo/myapp:1.0
```

### 12. **`docker-compose up`** / **`down`** – Manage multi-container apps (if using Compose)  
```bash
docker-compose up -d
docker-compose down
```

> *Bonus:* `docker system prune` – Clean unused containers, images, networks.
